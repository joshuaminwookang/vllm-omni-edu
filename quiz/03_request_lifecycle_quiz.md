# Quiz 03 — Request lifecycle and staged execution

Paired readings: `edu/docs/request_lifecycle.md`, `edu/docs/staged_execution.md`

## Multiple choice

1. Which entrypoint method normalizes prompts and sampling params before dispatching offline generation?
   - A. `Omni.generate()`
   - B. `StagePool.pick()`
   - C. `register_pipeline()`
   - D. `build_stage_runtime_overrides()`

2. Why does `_set_final_only_for_llm_stages()` matter?
   - A. It forces every diffusion output to become text.
   - B. It prevents ordinary offline generation from exposing token streams from intermediate LLM stages.
   - C. It disables request IDs.
   - D. It prevents stage config loading.

3. Which value determines how far a request should travel through a multi-stage graph?
   - A. `final_stage_id`
   - B. Git commit hash
   - C. local hostname
   - D. `requirements/common.txt`

4. `StageSubmissionMessage` is best described as:
   - A. A static Markdown heading
   - B. A queue message carrying request, prompt, sampling, final-stage, and timing information into the orchestrated runtime
   - C. A model weight tensor
   - D. A diffusion scheduler class

5. In a Qwen3-Omni-style speech pipeline, why might a text-only request stop earlier than an audio-output request?
   - A. Text-only requests have no request IDs.
   - B. The Thinker stage can be a final text output, while audio requires downstream Talker/Code2Wav stages.
   - C. Audio is always generated by the tokenizer.
   - D. The scheduler ignores output modality.

## Short answer

6. Trace a request from `Omni.generate()` to final `OmniRequestOutput`. Mention the entrypoint, async engine, orchestrator, stage pool, stage worker, and output message layers.

7. Explain what can go wrong if final-stage computation is incorrect.

8. Why are companion requests useful in AR→diffusion or CFG/KV workflows?

9. List six latency components in a staged omni request and explain why end-to-end latency alone is insufficient.

10. What lifecycle invariants must hold for request IDs and sampling params?

## Scenario questions

11. A request reaches Stage 0, produces a text result, and then unexpectedly runs Stage 1 and Stage 2 even though the user asked for text only. Where would you inspect first?

12. A request produces no final output, but stage metrics are emitted. What does this suggest about the difference between stage outputs and final user outputs?

13. A crash occurs while active requests exist. Why does the entrypoint attempt to abort active requests and close the engine?
